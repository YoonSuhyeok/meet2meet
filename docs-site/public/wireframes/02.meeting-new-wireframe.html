Annotations
1
인증 가드
useAuth()로 인증 확인. 비로그인 시 /login으로 리다이렉트. 로딩 중엔 스켈레톤 표시.
2
폼 필드 구성
• 제목: 필수, 최대 50자
• 설명: 선택, 최대 200자
• 장소: 선택, 자유 텍스트
모든 필드는 클라이언트 + 서버 양측 검증
3
Calendar 위젯
기존 Calendar 컴포넌트 재사용.
• minDate = 오늘
• maxDate = 오늘 + 30일
• 드래그 + Shift+클릭 범위 선택
• 선택된 날짜 요약 텍스트 하단 표시
→ Edge Case E2
4
시간 범위 설정
시작/끝 시간 <select> 드롭다운.
• 기본값: 00:00 ~ 24:00
• 30분 간격 옵션 (00:00 ~ 23:30)
• 끝 시간 > 시작 시간 검증
• generateTimeSlots(start, end, 30) 활용
5
TimeGrid 미리보기
날짜 + 시간 범위가 설정되면 자동으로 TimeGrid 미리보기 표시. 사용자에게 "참여자에게 이렇게 보입니다" 확인 목적.
읽기 전용 — 주최자는 시간 선택 없음. 이 그리드가 참여자용.
6
제출 버튼 & 유효성
활성화 조건: 제목 입력 + 날짜 1개+ 선택
비활성 시 opacity-40 + pointer-events-none
제출 중: 로딩 스피너 + 버튼 비활성화

Edge Cases & Policy
E1
비로그인 사용자 접근 → /login으로 리다이렉트 (로그인 후 /meeting/new로 복귀)
E2
날짜 14일 초과 선택 → 14일 제한 안내 토스트, 추가 선택 차단
E3
시간 범위 역전 (end < start) → 자동으로 end를 start+30분으로 조정
E4
제출 중 네트워크 오류 → 에러 배너 표시, 입력값 보존, 재시도 가능
E5
중복 제출 방지 → 제출 버튼 클릭 후 즉시 disabled + 로딩 스피너. 응답 받을 때까지 유지.
E6
세션 만료 중 제출 → 401 응답 → "세션이 만료되었습니다" 안내 후 /login 리다이렉트
E7
제목 50자 초과 입력 → maxLength로 차단 + 실시간 글자 수 카운터
E8
페이지 뒤로가기/새로고침 → 입력 데이터 없어짐 (별도 저장 없음). 복잡하면 추후 beforeunload 경고 추가.
E9
공유 링크 복사 실패 → navigator.clipboard 불가 시 fallback execCommand('copy'), 최종 실패 시 URL 텍스트 선택

Component Architecture
src/pages/meeting/new/
├── +Page.tsx            ← 메인 페이지 (폼 상태 관리)
│
├── MeetingForm.tsx     ← 폼 UI (제목/설명/장소 입력)
│   └── 유효성 검증 로직 포함
│
├── ShareModal.tsx      ← 성공 모달 (링크+코드 공유)
│   └── 클립보드 복사 로직 포함
│
재사용 위젯:
├── widgets/date-picker → Calendar
│   └── CalendarProps: { selected, onSelectionChange, minDate, maxDate }
│
├── widgets/time-picker → TimeGrid (미리보기용, read-only)
│   └── TimeGridProps: { dates, timeSlots, selected, onSelectionChange }
│
└── entities/meeting   → generateTimeSlots()
    └── 시간 범위 → 슬롯 생성 유틸리티
