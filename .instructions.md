# Meet2Meet — 프로젝트 규칙

## 기술 스택

- **프론트엔드 + BFF**: Vike + Hono + React 19 (Cloudflare Workers)
- **스타일링**: Tailwind CSS v4
- **빌드/배포**: Vite, Wrangler
- **린트/포맷**: Biome
- **패키지 매니저**: pnpm
- **타입 체크**: tsgo (TypeScript Go)
- **인증**: Hono JWT (HS256) + HttpOnly 쿠키, OAuth (Naver/Kakao/Google)

## 프로젝트 구조

```
src/
  +server.ts            # Hono 엔트리 (API 라우트 → Vike SSR)
  server/               # 서버 사이드 코드 (Hono 라우트)
  pages/                # Vike 페이지 (+Page.tsx, +data.ts 등)
  app/styles/           # 글로벌 CSS
  entities/             # 도메인 모델 (FSD)
  features/             # 기능 단위 (FSD)
  shared/               # 공용 UI, 유틸
  widgets/              # 복합 위젯 (FSD)
docs/
  ARCHITECTURE.md       # 시스템 전체 구조
  PROJECT_SETUP.md      # 환경 설정 가이드
  features/             # 기능별 기획 문서
```

## 기획 문서 규칙

- 전체 구조 문서는 `docs/ARCHITECTURE.md`에 작성
- 기능별 기획은 `docs/features/{기능명}.md`에 작성
- 기능 문서 템플릿:

```markdown
# {기능명}

## 목표
한 줄 설명

## 유저 스토리
- 사용자는 ~할 수 있다

## 화면 흐름
1. → 2. → 3.

## API 엔드포인트
| Method | Path | 설명 |

## 데이터 모델
타입 / 테이블 정의

## 구현 상태
- [x] Phase 1: ...
- [ ] Phase 2: ...
```

## 코드 컨벤션

- Biome 린트 규칙 준수 (`pnpm lint:fix`로 자동 수정)
- Vike `+` 접두사 파일은 페이지 모듈 전용. Stories 파일에 `+` 붙이지 않기
- 환경변수: 로컬은 `.dev.vars`, 배포는 `wrangler secret`
- 한국어 주석 사용
