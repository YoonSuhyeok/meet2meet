Edge Cases and Policy
E1
중복 발송 클릭 -> 같은 payload 해시로 30초 내 재요청 시 409 및 기존 reminderId 반환
E2
부분 발송 실패 -> 실패 대상만 retry 큐에 적재하고 상태는 partial_failed로 유지
E3
행사 시작 이후 Ack 제출 -> 410 응답, 읽기 전용 결과 화면으로 전환
E4
이미 불참 응답 후 변경 -> 행사 시작 전까지만 상태 변경 허용, 이력 로그 기록
E5
미응답자 과다 -> 임계치(30%) 초과 시 주최자에게 장소/시간 재안내 CTA 노출
